PHP 跨站脚本漏洞解析

Cross-Site Scripting (XSS) attacks are a type of injection, in which malicious scripts are injected into otherwise benign and trusted websites. XSS attacks occur when an attacker uses a web application to send malicious code, generally in the form of a browser side script, to a different end user. Flaws that allow these attacks to succeed are quite widespread and occur anywhere a web application uses input from a user within the output it generates without validating or encoding it.

这是 OWASP 对跨站脚本漏洞（Cross-Site Scripting）的描述。大意为“当恶意脚本被注入到原本安全并可信任的网站中，攻击者使用Web应用程序将恶意代码发送给其他用户时，便会触发跨站脚本漏洞”。

跨站脚本漏洞原称为 CSS（根据英文名称 Cross-Site Scripting 缩写），因为和层叠样式表（Cascading Style Sheets）重名，所以改称为 XSS 。

攻击者可以使用 XSS 漏洞将恶意脚本发送给毫无戒心的用户。最终用户的浏览器无法判断该脚本是否受信任，并执行该脚本。

漏洞详解

XSS 一般分为反射型、存储型、 DOM 型。DOM 型比较特殊。下面主要是围绕存储型和反射型进行整理分析。

当然我们把分类放在一边，先来梳理一下什么是 XSS 漏洞。在 OWASP 的定义基础上，再直白一点讲，XSS 就是网站将用户输入的内容输出到页面上，在这个过程中存在恶意代码并被浏览器执行。

其原理是由于Web应用程序对用户输入过滤不足，这时候攻击者就可以通过这些不足在网页上面注入一些比如盗取 cookie ，转账等等的 JavaScript 脚本恶意代码，当用户浏览这些网页时就会执行其中的恶意代码，从而就完成一个的完整的 XSS 攻击流程。

一句话，XSS就是在用户的浏览器中执行攻击者自己定制的脚本。

那么再说回来，什么是反射型 XSS ，存储型 XSS ？

反射型XSS又叫非持久型XSS，是用户访问到攻击者设置的恶意网站或经过设置的URL，在网页返回的内容中出现攻击者设置的XSS脚本代码，并被浏览器解析执行。反射的意思就是XSS攻击脚本被Web服务器反射回来给浏览器解析执行。

存储型XSS也叫持久性XSS，是攻击者将恶意脚本代码存储到Web服务器中并被解析，客户当访问到这网页时将受到攻击。

反射型 XSS 和存储型 XSS 就好像一个是足球，一个是铅球。足球（包含XSS的url）砸向草地，足球会弹起来（反射），而铅球（包含XSS的信息）砸向草地，则会留下一个坑（存储）。不砸足球，就不会反弹，而只要砸了一次铅球，坑就会一直存在。

虽然现在反射型 XSS 的受重视程度远不如存储型 XSS，但若说足球和铅球砸向我们的头，哪个痛？肯定还是头最痛。

漏洞场景

  1. 盗取各类用户帐号

  2. 控制网站服务器数据

  3. 盗窃所属网站公司的商业价值资料

  4. 非法转账

  5. 传播 XSS 跨站脚本蠕虫

  6. 结合其他漏洞进行综合漏洞攻击

  7. 提升攻击者权限

  8. 控制受害者机器向其他网站发起攻击

  9. 网站挂马
