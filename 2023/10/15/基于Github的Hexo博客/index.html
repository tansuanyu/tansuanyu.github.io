在配置其他环境之前，首先我们要知道 Hexo 项目文件夹下面我们要用到什么。

  • _config.yml
    • 项目的配置文件
  • public
    • 项目生成的静态文件就存在这里
  • scaffolds
    • 模版 文件夹。新建文章时，Hexo 会根据 scaffold 来建立文件。
  • source
    • 存放用户资源的地方。除 _posts 文件夹之外，开头命名为 _ (下划线)的文件 / 文件夹和隐藏的文件将会被忽略。Markdown 和 HTML 文件会被解析并放到 public 文件夹，而其他文件会被拷贝过去。
  • themes
    • 主题文件夹。Hexo 会根据主题来生成静态页面。

域名绑定

当然，<username>.github.io的域名不仅难记，也不美观。我们完全可以使用自己的域名进行访问。

域名的购买略去不表，在完成认证后，我们需要对域名进行 DNS 解析。添加两个 A 记录，绑定 IPv4 地址192.30.252.153和192.30.252.154，这是 Github pages 官网所说的IP地址。但我们每个人都解析到两个IP地址的话，那么Github怎么知道我的域名而不是其他人的域名呢？

这就需要我们在 Github pages 根目录上新建一个 CNAME 文件，在 CNAME 文件上写入你想绑定的域名。

而这又引出了另一个问题，我们每次部署文件时，都会把 CNAME 文件覆盖掉。为了避免每次都重新建立 CNAME 文件，我们可以让 CNAME 文件伴随着静态文件一起更新到 Github pages 仓库中。而 hexo 又会删除覆盖整个 public 文件夹，所以就需要在 public 文件夹下设定一个无权限删除但是可读的 CNAME 文件。即取消勾选所有的 CNAME 文件属性 -> 安全 -> 高级 -> 禁用继承 -> 主体 -> 高级权限 -> 删除 。

部署忽略选项

在我们安装插件后，博客文件夹中出现了 node_modules 文件夹和配置文件，有时 git 会选择将这些文件进行上传，这就需要我们手动在 git 中剔除掉不需要上传的部分，避免将 node_modules 文件夹和配置文件或是其他的一些不必要上传的文件部署到 Github pages 上。

我们只需要在 Blog 根目录上新建一个 .gitignore 文件，在其中输入我们想要忽略的文件夹或是文件即可。
